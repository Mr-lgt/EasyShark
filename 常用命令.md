# EasyTshark 项目常用命令

本文档整理了 EasyTshark 项目的编译、运行、调试及其他常用命令，适用于 Windows 环境。

## 编译命令

### 使用 CMake 构建 Debug 版本
```powershell
cd E:\MyProject\EasyTshark\build
cmake --build . --config Debug
```

### 使用 CMake 构建 Release 版本
```powershell
cd E:\MyProject\EasyTshark\build
cmake --build . --config Release
```

### 清理构建文件
```powershell
cd E:\MyProject\EasyTshark\build
rm -Recurse -Force CMakeFiles
del CMakeCache.txt
del cmake_install.cmake
del Makefile
del *.exe
```

## 运行命令

### 直接运行 Debug 版本
```powershell
cd E:\MyProject\EasyTshark\build
.\EasyTshark.exe
```

### 直接运行 Release 版本
```powershell
cd E:\MyProject\EasyTshark\build
.\EasyTshark.exe
```

## 调试命令

### 使用 MinGW GDB 调试
```powershell
cd E:\MyProject\EasyTshark\cmake-build-debug-mingw
gdb .\EasyTshark.exe
```

在 GDB 中可以使用以下命令：
- `run`: 运行程序
- `break 文件名:行号`: 设置断点
- `step`: 单步执行
- `next`: 单步跳过函数调用
- `print 变量名`: 打印变量值
- `continue`: 继续执行到下一个断点
- `quit`: 退出 GDB

### 使用 Visual Studio 调试
1. 打开 `E:\MyProject\EasyTshark\cmake-build-debug-visual-studio-64` 目录
2. 双击 `EasyTshark.sln` 打开 Visual Studio
3. 在 Visual Studio 中设置断点并按 F5 调试

## 其他常用命令

### 生成 CMake 配置（首次构建）
```powershell
mkdir -p E:\MyProject\EasyTshark\build
cd E:\MyProject\EasyTshark\build
cmake ..
```

### 生成编译数据库（用于代码分析工具）
```powershell
cd E:\MyProject\EasyTshark\build
cmake -DCMAKE_EXPORT_COMPILE_COMMANDS=ON ..
```

### 查看 CMake 变量
```powershell
cd E:\MyProject\EasyTshark\build
cmake -L
```

### 使用 Clang-Tidy 进行代码检查
```powershell
cd E:\MyProject\EasyTshark\build
clang-tidy -p . ..\task\TsharkManager.cpp
```

## 项目结构说明

- `E:\MyProject\EasyTshark\build`: 默认构建目录
- `E:\MyProject\EasyTshark\cmake-build-debug-mingw`: MinGW 调试构建目录
- `E:\MyProject\EasyTshark\cmake-build-debug-visual-studio-64`: Visual Studio 64位调试构建目录
- `E:\MyProject\EasyTshark\task`: 主要源代码目录
- `E:\MyProject\EasyTshark\utils`: 工具类目录
- `E:\MyProject\EasyTshark\third_library`: 第三方库目录

## 注意事项

1. 所有命令均在 Windows PowerShell 环境下测试通过
2. 执行命令前请确保已安装必要的工具链（CMake、GCC/MinGW 或 Visual Studio）
3. 首次构建前需执行 CMake 配置命令
4. 调试命令需要对应调试版本的可执行文件
